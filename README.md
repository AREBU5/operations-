# operations-
Operations Management SQL Analysis

This project demonstrates the use of SQL to analyze data from an operations management database. The queries focus on inventory management, customer order tracking, sales performance, and business reporting using joins, Common Table Expressions (CTEs), aggregate functions, and window functions.

Key analyses include:
📦 Displaying products alongside their current inventory levels.
⚠️ Identifying products that have fallen below their reorder threshold.
📊 Counting pending customer orders.
🚚 Retrieving shipped and pending sales orders.
💰 Calculating the total inventory value for each product.
🏆 Ranking customers by total revenue generated and identifying the top three customers.
SQL Concepts Demonstrated
JOIN (INNER JOIN, LEFT JOIN)
Common Table Expressions (CTEs)
Aggregate Functions (SUM, COUNT)
Window Functions (ROW_NUMBER)
Filtering with WHERE
Sorting with ORDER BY
Grouping with GROUP BY
Calculated Columns
Business Reporting Queries

This project simulates real-world operational reporting that could be used by warehouse managers, inventory analysts, and business intelligence teams to monitor stock levels, evaluate customer performance, and support data-driven decision-making.
